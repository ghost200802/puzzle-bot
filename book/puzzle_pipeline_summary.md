# Puzzle-Bot 拼图流水线全流程总结

本项目是一个自动化拼图求解器，支持两种运行模式：**手机拍照模式（phone）**和**机器人模式（robot）**。两种模式共享核心的矢量化、去重、匹配和求解逻辑，区别主要在前端的数据采集方式。以下以手机拍照模式为主线，从原始照片出发，逐步描述整个拼图求解流水线。

---

## 流水线总览

整个流水线分为 **7 个大步骤**：

| 步骤 | 名称 | 输入 | 输出 | 核心文件 |
|------|------|------|------|----------|
| 1 | 照片预处理 | 原始照片 | 灰度图 + 彩色图 | `src/common/preprocess.py` |
| 2 | 背景分割与拼图块提取 | 灰度图 + 彩色图 | 单个拼图块（二值 + 彩色） | `src/common/segment_phone.py`, `src/common/extract.py` |
| 3 | 矢量化（找角/找边） | 单个拼图块二值图 | 四条边的顶点序列 + 边/角信息 | `src/common/vector.py`, `src/common/sides.py` |
| 4 | 去重 | 所有矢量化后的拼图块 | 去重后的唯一拼图块集合 | `src/common/dedupe.py` |
| 5 | 建立连接图（匹配） | 去重后的拼图块集合 | 每个拼图块各边可匹配的其他块列表 | `src/common/connect.py`, `src/common/pieces.py` |
| 6 | 求解拼图 | 连接图 | 完整的拼图排列方案（棋盘） | `src/common/board.py` |
| 7 | 输出与后处理 | 求解结果 | 可视化网格 / 运动指令 / 松紧调整 | `src/common/output.py`, `src/common/move.py`, `src/common/spacing.py` |

**流水线入口文件：**
- 手机模式完整运行：`src/run_batch.py`
- 端到端示例脚本：`run_pipeline.py`, `run_e2e.py`
- 处理模块：`src/process.py`
- 求解模块：`src/solve.py`
- 全局配置：`src/common/config.py`

## 步骤 1：照片预处理

**目标：** 将手机拍摄的原始照片转换为标准化的灰度图和彩色图，消除不同拍摄条件带来的差异。

**子步骤：**

1. **EXIF 自动旋转** — 读取照片的 EXIF 方向信息，将照片旋转到正确方向，转换为 BGR 格式。
2. **色彩归一化（白平衡）** — 使用 Gray World 算法，将 BGR 三个通道的均值拉平到同一水平，消除色温偏差。
3. **CLAHE 对比度增强** — 在 LAB 色彩空间的 L 通道上应用限制对比度的自适应直方图均衡化，增强局部对比度。
4. **灰度转换** — 将归一化后的 BGR 图像转换为灰度图，供后续分割使用。

**对应文件：** `src/common/preprocess.py`

---

## 步骤 2：背景分割与拼图块提取

**目标：** 从照片中分离出背景和拼图块，将每个拼图块单独裁切出来。

**子步骤：**

### 2.1 背景亮度检测
- 采样图像四周边框区域的像素亮度中位数
- 判断背景是"亮色"还是"暗色"，决定后续二值化的极性

### 2.2 二值化分割（支持三种方法，可降级）
1. **自适应阈值法（推荐默认）** — 对灰度图先做高斯模糊，再使用高斯加权自适应阈值进行二值化；亮背景使用 `THRESH_BINARY_INV`，暗背景使用 `THRESH_BINARY`；之后进行形态学开闭运算去噪。
2. **Otsu 全局阈值法** — 使用 Otsu 算法自动选择全局阈值，同样根据背景极性选择正/反二值化，后接形态学处理。
3. **GrabCut 法（降级备选）** — 用于背景复杂的场景，将图像中心区域标记为前景，运行 GrabCut 迭代分割。
4. **降级回退策略** — 先尝试 adaptive，失败再尝试 otsu，最后尝试 grabcut，选择像素面积最合理的结果。

### 2.3 去除杂散像素
- 迭代移除孤立像素（在 3x3 邻域中邻居数 ≤ 2 的像素）
- 去除细小噪点和半岛状突起

### 2.4 连通域提取（找岛）
- 使用 `scipy.ndimage.label` 对二值图做连通域标记
- 过滤掉面积小于阈值的连通域
- 记录每个连通域的边界框、是否接触图像边缘等信息

### 2.5 拼图块裁切与尺寸归一化
- 从二值图和彩色图中裁切出每个连通域对应的区域
- 标记是否为完整块（未接触图像边缘）
- 将每个拼图块缩放到统一的目标尺寸（默认 500px），保持二值和彩色信息

**对应文件：**
- `src/common/segment_phone.py` — 背景分割
- `src/common/extract.py` — 拼图块提取
- `src/common/find_islands.py` — 连通域分析

## 步骤 3：矢量化（找角点、提取四条边）

**目标：** 将每个拼图块从像素表示转换为几何表示——精确找到 4 个角点和 4 条边的顶点序列。

**子步骤：**

### 3.1 提取边界像素
- 扫描二值图像，找出所有"内部是 1、至少一个邻居是 0"的像素，标记为边界像素

### 3.2 边界轮廓追踪（矢量化）
- 从左上角第一个边界像素开始
- 使用顺时针邻域搜索算法，沿边界逐像素行走
- 记录每一步的转向角度，直到回到起点形成闭合轮廓
- 计算轮廓的质心（centroid）和内心点（incenter）

### 3.3 角点候选检测
- 对轮廓上的每个顶点，分析其两侧方向向量的夹角
- 计算候选角点的多项评分指标：
  - **角宽度**：两侧向量夹角（理想 90°）
  - **偏心距**：角平分线与质心方向的偏差
  - **直线度**：两侧 spokes 的方向标准差
  - **曲线分数**：周围点的弯曲程度（区分真实角点和曲线）
  - **质心对称性**：角点关于质心的对称程度
- 过滤掉分数过高的候选点
- 对边界区域（bbox 的上下左右边缘）进行补充搜索，放宽角度阈值，确保不遗漏边缘角点

### 3.4 候选角点合并
- 距离过近的候选点合并，保留分数更低的

### 3.5 最优四角选择
- 对所有候选点两两配对，评估作为"对角线"的合理性：
  - 两个角点应大致在质心的对侧（径向角差接近 180°）
  - 两个角点到质心的距离应相近
  - 两个角点的朝向也应大致相对
- 再对两对"对角线"配对，评估作为"四角"的合理性：
  - 四个角应大致均匀分布在质心周围（每 90° 一个）
  - 四角到质心距离一致
  - 惩罚过于紧密的角点分布
- 从所有组合中选择综合分数最低的 4 个角点

### 3.6 提取四条边
- 根据四个角点在轮廓上的位置，将轮廓分割为 4 段
- 合并距离过近的相邻顶点
- 对每条边计算其**归一化面积偏差**（顶点到首尾连线的平均距离），判断是否为**平直边缘**（边缘块的外侧边）
- 验证几何合理性：对边应近似平行，邻边应近似正交，每块最多 2 条边，且两条边不能平行

### 3.7 角点精修（增强角点）
- 物理拼图块的角点容易因磕碰而变形，因此需要修正
- 对每个角点，沿其两条边各回退一小段距离（避免弯曲部分），取中间段的顶点
- 对这两段顶点分别做最小二乘拟合趋势线
- 两条趋势线的交点即为修正后的角点位置
- 用新角点替换原始角点，并更新两条边的端点

### 3.8 保存矢量化结果
- 为每条边生成 JSON 文件（`side_{piece_id}_{side_index}.json`），包含顶点、中心、是否为边缘、内心点等
- 生成 SVG 调试图
- 提取颜色特征（HSV 直方图），保存为 JSON

**对应文件：**
- `src/common/vector.py` — 核心矢量化逻辑（边界追踪、角点检测、边提取、角点精修）
- `src/common/sides.py` — 边的表示、重采样、旋转对齐、误差计算
- `src/common/util.py` — 几何工具函数（角度计算、趋势线、多边形操作等）

## 步骤 4：去重

**目标：** 多张照片可能拍到同一块拼图，需要识别并去除重复块，只保留唯一的拼图块集合。

**子步骤：**

### 4.1 加载所有拼图块
- 读取所有 `side_*.json` 文件，重建每块拼图的 4 条 `Side` 对象

### 4.2 粗筛（基于空间位置）
- 如果两块拼图在原始照片中的位置（photo_space_origin）距离小于阈值，才进行精确比较
- 这一步大幅减少需要精确比较的对数

### 4.3 精确几何比较
- 对可能重复的两块拼图，尝试 4 种旋转排列（因为不知道拍照时拼图块的方向）
- 对每种排列，检查 4 条边的角度是否对齐（角度差不超过阈值）
- 计算对应边的拼合误差（边重采样后逐点比较），累加得到总几何误差
- 取 4 种旋转中最小的总误差作为相似度分数
- 分数低于阈值的判定为重复

### 4.4 并查集分组
- 使用 Union-Find 数据结构将所有重复块归入同一组

### 4.5 选出代表
- 每组选择一个代表（第一个成员），其余标记为重复
- 将代表块的文件复制到去重目录

**对应文件：** `src/common/dedupe.py`

## 步骤 5：建立连接图（边匹配）

**目标：** 找出每块拼图的每条非边缘边能与哪些其他拼图块的哪些边拼合在一起。

**子步骤：**

### 5.1 加载所有去重后的拼图块
- 从去重目录读取所有拼图块的 4 条边数据
- 对每条边进行**重采样**（统一为 26 个等距顶点）
- 将每条边**旋转对齐到水平方向**（方便后续比较）
- 同时生成"翻转"版本（代表拼合时的互补形状）

### 5.2 两两边匹配
- 对每块拼图的每条非边缘边，与所有其他拼图块的所有非边缘边逐一比较
- 匹配流程：
  1. **长度检查** — 两条边的长度差异不超过 8%，否则跳过
  2. **形状误差计算** — 将一条边与另一条边的翻转版本对齐，计算逐点绝对误差的总和，除以边长进行归一化
  3. **平移补偿** — 计算两条边的平均偏移量，平移后再算一次误差，取两次中较小的
  4. **阈值过滤** — 误差不超过 3.5 的判定为可匹配

### 5.3 颜色辅助匹配（可选）
- 如果有颜色特征数据，在几何误差的基础上计算颜色相似度（HSV 直方图相关性）
- 将颜色相似度作为加权因子，降低几何误差（最多降低 30%）

### 5.4 保留最佳匹配
- 对每条边，只保留误差最小的若干匹配（误差不超过最佳匹配的 6 倍）
- 按误差升序排列

### 5.5 边缘检测容错
- 如果某条非边缘边没有任何匹配，将其标记为边缘边

### 5.6 保存连接图
- 将连接关系序列化为 JSON（`connectivity.json`）

**对应文件：**
- `src/common/connect.py` — 连接图构建主逻辑
- `src/common/pieces.py` — 拼图块数据结构
- `src/common/sides.py` — 边的误差计算
- `src/common/image_match.py` — 颜色匹配辅助

## 步骤 6：求解拼图

**目标：** 根据连接图，找到一种排列方式，使所有拼图块恰好填满 W×H 的网格。

**子步骤：**

### 6.1 识别角块和边缘块
- 统计每块拼图有多少条边是边缘（无匹配）
- 恰好 2 条相邻边缘的为**角块**（应有且仅有 4 块）
- 恰好 1 条边缘的为**边缘块**（总数应等于 2(W+H)-4）

### 6.2 选择起始角块
- 将 4 个角块按匹配总数降序排列
- 依次尝试每个角块作为起始点

### 6.3 优先队列搜索（螺旋放置）
- 将起始角块放置在棋盘的左上角 (0,0)，根据其边缘方向确定朝向
- 使用**最小堆（优先队列）**驱动搜索：
  - 每次从堆中取出误差最小的状态
  - 对当前位置，遍历当前拼图块在前进方向上能匹配的所有候选块
  - 对每个候选块，计算其在棋盘上的正确朝向（旋转 180° 对接），验证是否能放置：
    - 目标位置未被占用
    - 边缘约束满足（棋盘边界的块必须有对应的边缘边）
    - 已放置的邻居块必须在匹配列表中
  - 放置后推入堆中，移动到下一个位置
- 遇到棋盘边界或已放置的块时，自动转弯（右→下→左→上，螺旋前进）

### 6.4 成功判定
- 当棋盘上放置的拼图块数量等于 W×H 时，求解成功
- 返回完整的棋盘对象（包含每块的位置和朝向）

**对应文件：** `src/common/board.py`

## 步骤 7：输出与后处理

**目标：** 将求解结果输出为人类可读的形式，或在机器人模式下生成运动指令。

### 7.1 文本网格输出
- 生成 `solution_grid.txt`，用 ASCII 字符画出棋盘，每格显示拼图块编号和朝向箭头（^>v<）

### 7.2 标注目标图（可选）
- 如果提供了拼图盒子的参考照片，将其划分为 W×H 的网格
- 在每个格子上标注对应的拼图块编号，输出为 `annotated_target.png`

### 7.3 拼图块目录 HTML（可选）
- 生成 `piece_catalog.html`，按角块、边缘块、内部块分类展示所有拼图块

### 7.4 运动计算（机器人模式专用）
- 从左上角开始，沿棋盘螺旋遍历每个位置
- 对每个拼图块，计算：
  - **旋转角度**：使当前块的边与已放置邻居的边精确对接
  - **平移量**：取与所有已放置邻居的平移量的中点，实现最佳对齐
- 边界块使用虚拟邻居来保证棋盘的矩形对齐
- 输出每个拼图块从照片空间到目标空间的位置和旋转信息
- 生成整体棋盘的 SVG 可视化

### 7.5 松紧调整（机器人模式专用）
- 对每个拼图块的目标位置施加微小的线性偏移
- 根据拼图块在棋盘上的坐标，按固定比例增加间距（补偿物理拼合时的间隙）

**对应文件：**
- `src/common/output.py` — 文本/图像输出
- `src/common/move.py` — 运动计算
- `src/common/spacing.py` — 松紧调整

## 附加功能

### 实时识别（`src/common/real_time.py`）
- 通过摄像头实时捕获画面
- 对每一帧进行分割、提取、归一化
- 使用 ORB 特点匹配 + HSV 直方图对比，与已知的拼图块数据库进行比对
- 返回最可能的拼图块编号和置信度

### 目标图匹配（`src/common/target.py`）
- 将拼图盒子的参考照片分割为 W×H 的网格单元
- 对每个单元提取中心颜色和 HSV 直方图
- 将拼图块的颜色特征与网格单元进行匹配，辅助确定每块拼图的目标位置

## 数据流图

```
原始照片 (.jpg/.png)
    │
    ▼
[步骤1] 预处理 ──→ 灰度图 + 彩色图
    │
    ▼
[步骤2] 分割 + 提取 ──→ 单个拼图块 (二值 + 彩色)
    │
    ▼
[步骤3] 矢量化 ──→ 每块 4 条边的顶点 + 角点 + 边缘标记 (side_*.json + SVG)
    │
    ▼
[步骤4] 去重 ──→ 唯一拼图块集合
    │
    ▼
[步骤5] 建立连接图 ──→ 每块每边的候选匹配列表 (connectivity.json)
    │
    ▼
[步骤6] 求解 ──→ 完整棋盘 (Board 对象)
    │
    ▼
[步骤7] 输出 ──→ solution_grid.txt / annotated_target.png / 运动指令
```

## 关键算法与技术

| 领域 | 技术 |
|------|------|
| 图像分割 | 自适应阈值、Otsu、GrabCut、形态学运算 |
| 连通域分析 | scipy.ndimage.label、连通域过滤 |
| 轮廓追踪 | 顺时针邻域搜索、角度累积 |
| 角点检测 | 多指标评分 + 组合优化选择最优四角 |
| 角点精修 | 最小二乘趋势线拟合 + 直线交点 |
| 边匹配 | 多项式重采样 + 翻转对齐 + 逐点误差 |
| 去重 | Union-Find + 几何相似度 |
| 求解 | 优先队列（最小堆）+ 螺旋放置 + 约束验证 |
| 颜色匹配 | HSV 直方图相关性、ORB 特征匹配 |
| 几何工具 | Shapely 多边形操作、Ramer-Douglas-Peucker 简化 |
