# 拼图助手项目 — 总体架构与流程设计

## 一、项目定位

将 `puzzle-bot`（机器人拼图求解器）改造为**手机拍照辅助人工拼图**的系统。
用户通过手机拍摄拼图块照片 + 目标图，系统自动识别每块拼图的位置，在拼图过程中提供实时指引。

## 三、输入要求

### 阶段 A 输入
| 输入 | 说明 |
|------|------|
| 目标图 | 拼图包装盒照片（包装盒上的完整图） |
| 拼图块照片 | 手机拍摄的多张照片，每张可能含多个拼图块 |
| 拼图规格 | 列数 × 行数（如 20×15 = 300片） |

### 阶段 A 照片特点（需处理）
- 一张照片可能包含**多个拼图块**
- 不同照片的拍摄**角度、俯仰、距离**可能不同
- 可能有**重复**的拼图块（同一块出现在多张照片中）
- 背景不统一（桌面、纸板等）
- 可能有畸变和透视变形

### 阶段 B 输入
| 输入 | 说明 |
|------|------|
| 实时照片 | 用户拍摄当前手中的 1~N 块拼图块 |

### 缺失处理
- 用户声明总片数（如 300 片）
- 如果预处理只提取到 280 片 → 正常构建，标记 20 片缺失
- 缺失位置留空，后续实时阶段可补充

## 四、数据流与存储

### 拼图数据库（核心中间产物）
```
puzzle_database/
├── pieces/
│   ├── piece_001/
│   │   ├── binary.bmp          # 归一化后的二值图
│   │   ├── color.png           # 归一化后的彩色图
│   │   ├── sides.json          # 四条边的向量化数据
│   │   ├── features.json       # 颜色/纹理特征
│   │   └── thumbnail.png       # 缩略图（用于输出）
│   ├── piece_002/
│   └── ...
├── target/
│   ├── target_image.png        # 原始目标图
│   ├── target_grid.json        # 网格分割数据
│   └── target_cells/           # 每个格子的特征
│       ├── cell_0_0.json
│       └── ...
├── connectivity/
│   └── connectivity.json       # 边匹配关系图
├── solution/
│   ├── solution.json           # 求解结果
│   └── solution_grid.txt       # 文字版解方案
└── database_meta.json          # 元数据（总片数、已识别片数等）
```

## 五、从现有项目复用的代码

### 完全复用（不改或极小改动）
| 模块 | 文件 | 复用内容 |
|------|------|---------|
| 几何工具 | `util.py` | 距离、角度、旋转、重采样、误差计算 |
| 边数据 | `sides.py` | Side 类、`error_when_fit_with()` |
| 拼图块模型 | `pieces.py` | Piece 数据模型 |
| 轮廓追踪 | `vector.py` | `find_border_raster()`, `vectorize()`, `find_four_corners()`, `extract_four_sides()`, `enhance_corners()` |
| 求解器 | `board.py` | Board 类、DFS 求解、`can_place()` |
| 连通域 | `find_islands.py` | `extract_islands()`, `remove_stragglers()` |

### 需要改造的模块
| 模块 | 文件 | 改造内容 |
|------|------|---------|
| 配置 | `config.py` | 新增手机模式参数、移除机器人参数 |
| 入口 | `run_batch.py` | 新命令行参数、两阶段入口 |
| 流程 | `process.py` | 新流水线 |
| 提取 | `extract.py` | 适配新分割，保留颜色 |
| 去重 | `dedupe.py` | 基于图像特征的去重 |
| 匹配 | `connect.py` | 集成图像匹配评分 |

### 需要新增的模块
| 模块 | 文件 | 功能 |
|------|------|------|
| 预处理 | `preprocess.py` | EXIF旋转、透视校正、颜色归一化 |
| 分割 | `segment_phone.py` | 自适应背景分割 |
| 目标图 | `target.py` | 目标图处理与网格特征 |
| 图像匹配 | `image_match.py` | 颜色/纹理匹配 |
| 输出 | `output.py` | 人类可读输出生成 |
| 实时识别 | `realtime.py` | 在线阶段的拼图块识别 |
| 数据库 | `database.py` | 拼图数据库管理 |

### 废弃的模块
| 文件 | 原因 |
|------|------|
| `bmp.py` | 被新 `segment_phone.py` 替代 |
| `move.py` | 机器人运动指令不再需要 |
| `spacing.py` | 机器人间距调整不再需要 |
